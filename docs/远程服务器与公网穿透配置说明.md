### 3.5 远程 sshd 配置要点

配置文件路径：`C:\ProgramData\ssh\sshd_config`

- `Port 52222` — 监听端口
- `PubkeyAuthentication yes` — 启用公钥认证
- `PasswordAuthentication no` — 禁用密码登录
- `UseDNS no` — 禁用 DNS 反向解析
- 防火墙已放行 52222 端口
- VMware 相关服务已禁用（避免端口冲突）

### 6.1 基本信息

| 项目 | 详情 |
| --- | --- |

| 固定域名 | `resistive-diotic-jolie.ngrok-free.dev` |

| ngrok 管理面板 | https://dashboard.ngrok.com |

### 6.2 使用方法（通用示例）

> 以下以 Python HTTP 服务为例，实际端口以你的 Web 服务为准（小账本 Web 后台用的是 **5000** 端口，见第 7 节）。

步骤 1：在远程电脑上启动 Web 服务：
```bash
# 示例：Python HTTP 服务用 8080 端口
python -m http.server 8080 --directory 你的网页目录 --bind 0.0.0.0
```

步骤 2：启动 ngrok 穿透（**端口必须与 Web 服务一致**）：
```bash
# 端口号要和上面的 Web 服务端口一样
C:\Users\1\Desktop\ngrok\ngrok.exe http 8080 --domain=resistive-diotic-jolie.ngrok-free.dev
```

步骤 3：外部用户通过以下地址访问：
```
https://resistive-diotic-jolie.ngrok-free.dev
```

### 6.3 注意事项

- 启动 ngrok 时必须加 `--domain` 参数，否则域名会随机变化
- **ngrok 端口必须与 Web 服务监听端口一致**，否则 ngrok 会返回 502/503
- 免费版首次访问会有 ngrok 确认页面，点击 "Visit Site" 即可
- 关闭 ngrok 后公网地址失效，重新启动即恢复
- 不加 `--domain` 参数时，每次重启 ngrok 域名都会变化

## 7. Web 后台管理系统部署（小账本）

### 7.1 远程部署信息

| 项目 | 详情 |
| --- | --- |
| 项目路径 | `C:\Users\1\Desktop\web-admin` |
| 技术栈 | Node.js + Express + MySQL |
| 数据库 | MySQL 8.0，数据库名 `accountbook` |
| 服务端口 | **5000**（`server.js` 中的 `PORT`） |
| ngrok 穿透 | `ngrok.exe http 5000 --domain=...` |
| 公网地址 | `https://resistive-diotic-jolie.ngrok-free.dev` |
| 启动脚本 | `C:\Users\1\Desktop\web-admin\start.bat` |
| 任务计划名 | `WebAdmin`（用于 schtasks 后台启动） |

### 7.2 start.bat 启动脚本内容

```batch
@echo off
cd /d C:\Users\1\Desktop\web-admin
start "WebAdmin-Node" /B node server.js
timeout /t 3 /nobreak >nul
start "WebAdmin-Ngrok" /B C:\Users\1\Desktop\ngrok\ngrok.exe http 5000 --domain=resistive-diotic-jolie.ngrok-free.dev
echo Started node + ngrok
```

> 注意：脚本中 ngrok 端口（5000）必须与 `server.js` 的 `PORT` 一致。

### 7.3 部署步骤

**首次部署（在本地电脑执行）：**

```bash
# 1. 从本地 SCP 上传项目到远程（注意 -P 大写，端口 52222）
scp -P 52222 -r c:\Users\Administrator\Desktop\AccountingApp\web-admin 1@100.114.218.70:C:\Users\1\Desktop\web-admin

# 2. SSH 到远程初始化数据库（只需首次执行）
ssh remote "cd C:\Users\1\Desktop\web-admin; node init-db.js"

# 3. 创建后台任务并启动（/ST 00:00 会有时间警告，不影响 /Run）
ssh remote "schtasks /Create /TN WebAdmin /TR 'C:\Users\1\Desktop\web-admin\start.bat' /SC ONCE /ST 00:00 /F /RU 1; schtasks /Run /TN WebAdmin"

# 4. 等待 10 秒后验证端口
ssh remote "Start-Sleep 10; netstat -ano | Select-String ':5000|:4040'"
# 应该看到 5000 和 4040 都在 LISTENING
```

**更新代码后重新部署：**

```bash
# 1. 上传修改的文件（注意路径要对应远程目录结构）
scp -P 52222 web-admin\server.js 1@100.114.218.70:C:\Users\1\Desktop\web-admin\server.js
scp -P 52222 web-admin\routes\api.js 1@100.114.218.70:C:\Users\1\Desktop\web-admin\routes\api.js

# 2. 停止旧进程并重启
ssh remote "taskkill /IM node.exe /F; taskkill /IM ngrok.exe /F; Start-Sleep 2; schtasks /Run /TN WebAdmin"
```

### 7.4 踩坑记录

> ⚠️ **SSH 端口**：必须用 `ssh remote`（别名）或 `-p 52222`，默认端口 22 会超时。

> ⚠️ **远程 PowerShell 不支持 `&&`**：远程是 Windows PowerShell，多命令用 `;` 分隔，不能用 `&&`。

> ⚠️ **Start-Process 创建的进程不持久**：通过 SSH 用 `Start-Process` 启动的 node/ngrok 进程，SSH 断开后虽然进程存在于 `Get-Process`，但实际无法正常工作（端口虽然显示 LISTENING 但连接会失败）。**正确做法是用 `schtasks`（任务计划）或 `start.bat`**。

> ⚠️ **ngrok 端口必须与 Web 服务一致**：`server.js` 用 5000 端口,  ngrok 就必须 `http 5000`。端口不一致会返回 502/503。

> ⚠️ **端口冲突**：远程机器 8080 端口可能被其他服务占用。先用 `netstat -ano | Select-String ':端口号'` 检查端口是否可用。

> ⚠️ **MySQL 密码**：`db.js` 中配置的密码是 `root123456`，确保远程 MySQL 的 root 密码也是这个。
