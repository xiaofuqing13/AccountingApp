# 启动页视觉优化（2026-02-20）

## 目标
- 解决启动页观感单薄、图标发黑的问题。
- Android 12+ 使用系统 SplashScreen，保证首帧一致性。
- 应用内启动页保留品牌风格，并支持按钮手动进入。

## 本次修改
- `app/src/main/res/layout/activity_splash.xml`
  - 重建启动页布局，文案全部改为资源引用，避免硬编码乱码。
  - 保留中央卡片 + 渐变背景，优化可读性。
- `app/src/main/res/values/strings_splash.xml`
  - 新增启动页专用文案：标题、副标题、按钮、底部文案。
- `app/src/main/res/values-v31/themes.xml`
  - `Theme.AccountBook.Splash` 改为 `Theme.SplashScreen`（AndroidX）。
  - 配置 `windowSplashScreenBackground`、`windowSplashScreenAnimatedIcon`、`windowSplashScreenIconBackgroundColor`、`postSplashScreenTheme`。
- `app/src/main/res/drawable/ic_splash_brand.xml`
  - 新增首帧专用品牌图标（固定粉色），避免部分设备回退为黑色心形。
- `app/src/main/java/com/loveapp/accountbook/ui/splash/SplashActivity.kt`
  - 在 `onCreate` 前调用 `installSplashScreen()`，让系统首帧和应用内启动页无缝衔接。
- `app/build.gradle.kts`
  - 新增依赖：`androidx.core:core-splashscreen:1.0.1`。

## 关键效果
- 系统首帧不再使用默认黑色图标。
- 启动页文案显示正常，不再出现乱码。
- 启动后过渡到主页更加自然。

## 验证
- 命令：`./gradlew.bat :app:compileDebugKotlin --offline`
- 结果：通过（无编译错误）。
