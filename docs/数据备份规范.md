# 数据备份规范

## 存储位置

数据文件固定存放在手机公共存储目录：

```
/storage/emulated/0/Documents/我们的小账本/我们的小账本.xlsx
```

该路径属于公共 Documents 目录，**卸载APP不会删除数据**，重新安装后自动读取。

### 权限降级策略

如果用户拒绝授予"所有文件访问"权限，APP 自动 fallback 到应用私有目录：

```
Android/data/com.loveapp.accountbook/files/DataManager/我们的小账本.xlsx
```

此路径下数据在卸载APP时会被清除，但不影响正常使用。

## 自动迁移

旧版APP数据存放在应用私有目录，新版首次访问数据时自动检测并迁移到公共目录：

1. 检测公共目录是否可写
2. 如果公共目录没有数据但旧路径有 → 自动复制到公共目录
3. 迁移成功后自动删除旧路径文件，避免占用双倍空间
4. 路径结果会被缓存，后续IO操作不再重复检测

## 文件格式（v2）

导出的 `.xlsx` 文件包含以下 5 个 Sheet：

| Sheet名称 | 列定义 | 说明 |
|-----------|--------|------|
| 记账 | 日期 / 类型 / 分类 / 金额 / 备注 | 5列，日期格式 yyyy-MM-dd |
| 日记 | 日期 / 标题 / 内容 / 天气 / 心情 / 位置 | 6列 |
| 会议纪要 | 日期 / 主题 / 开始时间 / 结束时间 / 地点 / 参会人 / 内容 / 待办事项 / 标签 | 9列 |
| 分类配置 | 分类名称 / 类型(支出/收入) / 图标 | 3列，自定义分类 |
| 元信息 | 键 / 值 | 格式版本号、应用名称、导出时间 |

### 导出文件命名

导出文件自动带时间戳：`记账备份_20260219_153000.xlsx`，多次导出不会互相覆盖。

## 版本兼容（智能导入）

导入时自动识别文件版本，无需用户区分：

| 判断条件 | 版本 | 处理方式 |
|---------|------|---------|
| 有"元信息"Sheet 或 Sheet名为"记账" | v2（当前版本） | 直接按列读取 |
| Sheet名为"记账明细" | 旧版 | 自动映射列顺序 |
| 记账表头第5列为"账户" | 旧版6列格式 | 跳过账户列，取第6列为备注 |
| 日记表头第2列为"天气" | 旧版日记格式 | 列顺序从 日期/天气/心情/标题/内容 映射到新版 |
| 会议纪要列数 ≤ 7 | 旧版会议格式 | 日期截取前10位，补齐开始/结束时间和标签为空 |

## 去重规则

| 数据类型 | 去重键 |
|---------|--------|
| 记账 | 日期 + 类型 + 分类 + 金额 + 备注 |
| 日记 | 日期 + 标题 |
| 会议 | 日期 + 主题 |
| 分类配置 | 分类名称 + 类型 |

空白行（日期为空或类型为空）自动跳过。

## 权限要求

- Android 11+（API 30）需要"所有文件访问"权限（`MANAGE_EXTERNAL_STORAGE`）
- 首次启动弹出对话框引导授权，用户可选择"暂不授权"
- 拒绝授权不影响APP使用，仅影响数据持久化位置

## 更新APP流程

1. 直接安装新版APK覆盖旧版
2. 数据文件在 Documents 公共目录，不受卸载/更新影响
3. 打开APP即可看到之前的全部数据，无需导入
