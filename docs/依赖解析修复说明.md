# 依赖解析修复说明

## 问题
- 构建时报错：`Could not find com.chauthai.swipereveallayout:swipe-reveal-layout:1.4.1`

## 处理
- 将依赖坐标统一为可用版本：
  - `com.github.chthai64:swipereveallayout:1.4.0`
- 处理 Duplicate class（`androidx.core` 与 `support-v4` 冲突）：
  - 在 `app/build.gradle.kts` 对 `swipereveallayout` 增加：
  - `exclude(group = "com.android.support", module = "support-v4")`
- 在两套 settings 中都加入 JitPack 仓库：
  - `settings.gradle.kts`
  - `app/settings.gradle.kts`
- 统一两套 wrapper 版本，避免子目录构建不一致：
  - `gradle/wrapper/gradle-wrapper.properties`
  - `app/gradle/wrapper/gradle-wrapper.properties`
  - 均为 `gradle-9.1.0-bin.zip`
- 同步 `app/gradle.properties` 与根工程兼容项，保证在 `app` 目录单独执行构建也可通过。

## 验证命令
- 根目录：
  - `.\gradlew.bat :app:checkDebugAarMetadata`
- app 子目录：
  - `.\gradlew.bat checkDebugAarMetadata`

两者均已验证通过。
