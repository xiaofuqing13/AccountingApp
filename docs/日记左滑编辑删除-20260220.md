# 日记左滑编辑删除改造（2026-02-20）

## 改造目标
- 将日记列表从“卡片常驻编辑/删除按钮”切换为“左滑触发编辑/删除”。

## 交互规则
- 日记列表项向左滑动：在卡片右侧露出“修改 / 删除”按钮。
- 点击卡片：查看详情。
- 长按卡片：不再作为编辑/删除入口，统一改为左滑触发。

## 代码变更
- `app/src/main/java/com/loveapp/accountbook/ui/diary/DiaryListFragment.kt`
  - 使用 `ItemTouchHelper` 实现左滑并限制滑动距离。
  - 左滑后固定露出按钮区，不再弹出二级操作弹窗。
  - 增加开合动画、右滑收起（轻拉即可回收）与动态阈值，滑动手感更平滑。
- `app/src/main/java/com/loveapp/accountbook/ui/adapter/DiaryAdapter.kt`
  - 新增 `onEditClick/onDeleteClick` 回调，绑定左滑露出的按钮点击事件。
  - 增加展开项状态管理，确保同一时间仅一条卡片展开。
- `app/src/main/res/layout/item_diary.xml`
  - 新增卡片底层操作区（`btn_swipe_edit`、`btn_swipe_delete`），前景卡片左滑露出按钮；操作区样式简化，避免圆角边缘不顺滑。

## 验证
- 编译通过：`:app:compileDebugKotlin`
- 安装通过：`:app:installDebug`
- UI 树验证：日记列表中不再存在 `btn_edit/btn_delete` 常驻控件。

## 2026-02-20 二次修复（左滑收回与露角）
- 修复左滑后难以收回：`clearView` 按当前位置强制归位（超过半宽保持展开，否则回弹关闭）。
- 增加右滑回收即时响应：`ACTION_MOVE` 阶段满足水平位移即触发收回，避免卡在半开状态。
- 修复右侧露角：新增 `bg_diary_swipe_actions.xml`，操作区使用与卡片一致的圆角背景并开启裁剪。
- 视觉细节优化：移除操作按钮上下 margin，避免边缘残留色块。
- 同步修复：清理日记详情位置文案乱码与适配器异常兜底文案乱码。
- 收回稳定性加强：新增 `forceCloseUntilNextClear` 关闭优先标记，防止回收后被 `clearView` 误判再次展开。
- 右滑门槛下调：已展开卡片的 `getSwipeThreshold` 降为 `0.06f`，轻滑即可回收。
- 视觉还原升级：日记左滑操作区改为 iOS 风格“圆形图标 + 下方文字”，替代原先整块矩形按钮。
- 样式资源新增：`ic_swipe_edit`、`ic_swipe_delete`、`bg_swipe_action_circle_edit`、`bg_swipe_action_circle_delete`。
- 交互区域尺寸：`diary_swipe_action_width` 调整为 `86dp`，确保图标与文案留白接近系统备忘录观感。
- 点击命中修复：左滑展开后，按钮区命中改为基于“当前展开项真实右边界+操作区宽度”计算，避免 `RecyclerView` padding 导致点击被误判为卡片主体。
- 触控区域对齐：去掉操作区左右 padding，确保展示宽度与滑动宽度一致，避免编辑按钮局部点不到。
- 回收误触修复：按钮区按下时不再参与右滑回收判定，优先触发按钮点击。
- 回收阈值从 `3dp` 提升到 `10dp`，降低点按微抖动导致的误判回收。
- 紧急止血：移除 `attachSwipeCloseFallback` 触摸拦截（该逻辑会与按钮点击冲突，导致“点不了/自动回收”）。
- 左滑交互改为单通道：仅通过 `ItemTouchHelper + clearView` 决定开合状态，避免多套状态机互相打架。
- `getSwipeThreshold` 设为 `1f`，不再依赖 `onSwiped` 回调切状态，减少“滑不回/点击无效”概率。
- 点击误判滑动修复：新增操作区手势保护。已展开项在操作区按下时，临时禁用该条目的 `ItemTouchHelper` 滑动，仅保留按钮点击。
- 二次加固：按钮区按下时启用全局滑动锁（`blockAllSwipesForActionTap`）并 `requestDisallowInterceptTouchEvent(true)`，彻底防止点击被识别为滑动。
- 稳定性优先策略：存在已展开项时全局禁用滑动（`getSwipeDirs=0`），避免 `ItemTouchHelper` 抢占按钮点击手势。
- 触摸优先级修复：编辑/删除按钮新增 `OnTouchListener`，按下即禁止父容器拦截，抬起强制 `performClick()`，防止点按被识别成滑动。
- 操作按钮容器显式设置 `clickable/focusable=true`，提升命中稳定性。
