## 崩溃修复（点击“日记”闪退）
- 现象：用户点击底部导航“日记”后，页面直接退出。
- 原因：日记列表加载与渲染路径缺少异常兜底，任意数据/IO异常会直接抛到主线程导致崩溃。
- 修复点：
  - `DiaryViewModel.kt`：为 `load/search/add/update/delete` 增加 `runCatching`，失败时回退安全状态并抛出可观察错误消息。
  - `DiaryListFragment.kt`：监听 `errorMessage`，在页面内提示错误，避免无提示闪退。
  - `DiaryAdapter.kt`：`onBindViewHolder` 增加单条数据绑定保护，坏数据只影响该条目，不影响整页。
- 验证：
  - `./gradlew.bat :app:assembleDebug` 构建通过。

## 根因修复（SwipeRevealLayout 运行时类缺失）
- 现象：点击底部“日记”后立即崩溃。
- 实际堆栈：`NoClassDefFoundError: android.support.v4.widget.ViewDragHelper$Callback`，触发点在 `DiaryAdapter.onCreateViewHolder`（加载 `SwipeRevealLayout`）。
- 原因：`com.github.chthai64:swipereveallayout:1.4.0` 使用旧 `android.support.*` 路径，当前工程未启用 Jetifier，运行时找不到该类。
- 处理：在 `gradle.properties` 与 `app/gradle.properties` 启用 `android.enableJetifier=true`，让依赖自动迁移到 AndroidX 类路径。
- 复测结果：安装新 APK 后自动点击“日记”可正常进入列表页，无 `FATAL EXCEPTION`。
