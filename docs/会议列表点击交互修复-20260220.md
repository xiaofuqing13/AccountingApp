# 会议列表点击交互修复（2026-02-20）

## 问题现象
- 会议列表页中，用户点击会议卡片主体（标题/地点/参会人等）无响应，只有日期区域可点击，导致“会议里的东西无法点击”的体验问题。

## 修复内容
- 为会议卡片整项增加单击事件绑定（不再仅限日期视图）。
- 单击会议卡片时弹出会议详情弹窗，展示日期、时间、地点、参会人、标签、会议内容、待办事项。
- 会议详情弹窗标题改为带边框高亮样式，增强视觉突出效果。
- 会议详情中“日期：”“时间：”前缀做加粗+主题色高亮，信息层级更清晰。
- 会议详情中其余前缀（地点/参会人/标签/会议内容/待办事项）也统一做加粗+主题色高亮。
- 会议列表支持左滑卡片露出“编辑/删除”按钮，交互与日记列表保持一致。
- 移除会议卡片长按编辑/删除入口，统一从左滑按钮进入。
- 左滑按钮为右侧横向双按钮（编辑/删除），并在滑动时增加卡片轻微缩放，减少突兀感。
- 保留原有日期点击彩蛋逻辑（2月14日触发情话彩蛋）。
