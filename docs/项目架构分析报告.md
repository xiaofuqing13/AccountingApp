# 项目结构与核心架构分析报告

## 一、项目概述
「我们的小账本」是一款以情侣为主题的 Android 原生记账应用，包含记账管理、情侣日记、会议纪要三大核心功能以及25个以上的隐藏甜蜜彩蛋。项目整体风格强调粉色少女系，并深度定制了相关的 UI 与交互。

## 二、技术架构
- **编程语言**：Kotlin 1.9.22
- **UI框架**：XML 布局体系 + Material Design 3（未引用 Compose）
- **架构模式**：MVVM
- **导航控制**：Jetpack Navigation（单 Activity 多 Fragment）
- **数据持久化**：使用 Apache POI 读写 `.xlsx` 文件（单文件数据库）
- **异步处理**：Kotlin Coroutines (Dispatchers.IO)
- **最低兼容**：Android 8.0 (API 26)

## 四、核心业务模块说明

### 1. 数据层设计 (Excel 作为数据库)
没有采用常规的 SQLite 或 Room 数据库，而是利用 `ExcelRepository` 维护在 `Documents/我们的小账本/我们的小账本.xlsx` 这个文件中。
- 包含四个工作表：记账、日记、会议纪要、分类配置。
- 每个 ViewModel 的读写均打开并重新保存该文件。
- 设计了 `importSmart` 和 `exportShareable` 方法，可实现不同版本格式的自动兼容和全量分享。

### 2. UI交互与特色设计
- **MVVM与状态共享**：各个业务模块共享 Activity/Fragment 层级的 ViewModel 进行状态通讯。
- **自定义情感元素**：首页的恋爱计时器（从 2025-02-14 开始算起），智能时间问候语，以及分布在输入框、金额判定等各个环节的 25 种以上的交互彩蛋。
- **UI统一性**：强调全局视觉规范（主色调粉色 #E8729A 等），图标采用统一方案。后期更新中对日记左滑等互动进行了优化（见 docs 文件夹）。

## 五、总结
该项目在功能形态上相当完备，架构清晰（标准 Android MVVM 结构）。由于采用 Excel 作为单文件数据库，在数据迁移和可视化分享上有得天独厚的优势。目前项目脉络我已经基本掌握，接下来随时准备为您进行功能迭代或问题修复。
